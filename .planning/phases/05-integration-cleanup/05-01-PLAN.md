    Insert the following code block after line 36 (`const stress = ...`) and before line 38 (`if (!readiness && ...`):

    ```javascript
    // INT-02: Detect auth errors before the all-null guard.
    // Auth failures (401/403/no-token) cause all four ouraGet calls to reject.
    // Without this check, auth errors are indistinguishable from sync delays.
    const AUTH_ERRORS = ['AUTH_EXPIRED', 'AUTH_FORBIDDEN', 'NOT_AUTHENTICATED', 'REFRESH_FAILED'];
    const authError = [readinessResult, sleepResult, activityResult, stressResult]
      .filter(r => r.status === 'rejected')
      .map(r => r.reason?.message)
      .find(msg => AUTH_ERRORS.includes(msg));

    if (authError) {
      if (authError === 'AUTH_EXPIRED' || authError === 'REFRESH_FAILED') {
        process.stderr.write('Authentication expired. Run /oura auth to re-authenticate.\n');
      } else if (authError === 'AUTH_FORBIDDEN') {
        process.stderr.write('Access denied. Run /oura auth to re-authenticate with all required scopes.\n');
      } else {
        process.stderr.write('Not authenticated. Run /oura auth to connect your Oura account.\n');
      }
      process.exit(1);
    }
    ```

    CRITICAL: The existing `extractRecord` function must NOT be changed. It must continue to return `null` for DATA_NOT_SYNCED rejections so that individual unsynced sections are silently omitted. The auth check fires BEFORE the all-null guard, so the "Today's data hasn't synced yet" message only fires when rejections are non-auth.

    The existing all-null guard (`if (!readiness && !sleep && !activity && !stress)`) and the sync-delay message remain unchanged.

    This addresses ERR-02 (auth errors trigger clear re-auth instructions) and DASH-03 (dashboard gracefully handles missing data — auth errors are no longer misreported as sync delays).
