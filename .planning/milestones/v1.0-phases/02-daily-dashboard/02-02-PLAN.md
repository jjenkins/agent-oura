  <action>
Create `.claude/skills/oura/scripts/profile.mjs` as an ESM script. The script must:

**Imports:**
- `import { ouraGetWithRetry } from './client.mjs';` -- profile uses ouraGetWithRetry because there is no per-section collapse logic needed; errors should propagate as user-friendly messages.

**Parallel fetch:**
- Fetch both endpoints in parallel using `Promise.all()` (not allSettled -- both are required for profile, if either fails the whole command should error):
  - `ouraGetWithRetry('/usercollection/personal_info')` -- no date params needed; returns a direct object (NOT wrapped in `data:[]`)
  - `ouraGetWithRetry('/usercollection/ring_configuration')` -- no date params; returns `{ data: [...] }`

**Ring configuration extraction (Research Pitfall 4):**
- `ring_configuration` returns `{ data: [record1, record2, ...] }` -- user may have multiple ring setups
- Sort by `set_up_at` ascending and take the LAST record (most recent): `ringConfig.data.sort((a, b) => new Date(a.set_up_at) - new Date(b.set_up_at)).at(-1)`
- If `data` is empty or undefined, set ring to `null`

**Output format (D-13, D-14):**
Build output lines:

```
=== Oura Profile ===

Personal Info:
  Email: ${personalInfo.email}
  Age: ${personalInfo.age}
  Height: ${personalInfo.height} cm
  Weight: ${personalInfo.weight} kg
  Biological sex: ${personalInfo.biological_sex}

Ring Configuration:
  Hardware: ${ring.hardware_type}
  Color: ${ring.color}
  Design: ${ring.design}
  Firmware: ${ring.firmware_version}
  Size: ${ring.size}
  Setup date: ${ring.set_up_at.split('T')[0]}
```

- Only output each Personal Info field if the value is not null/undefined (D-13 says show everything the API returns, but gracefully handle nulls)
- Only output the Ring Configuration section if a ring record exists
- For `set_up_at`, split on 'T' to show just the date portion (YYYY-MM-DD)

**Error handling:**
- Wrap in try/catch. On error, output `error.message` to stderr and exit with code 1.
- Since `ouraGetWithRetry` already converts errors to human-readable strings, the catch block just needs to print and exit.

**Output to stdout:**
- `process.stdout.write(lines.join('\n') + '\n');`

**What NOT to do:**
- Do NOT import chalk or cli-table3
- Do NOT add date parameters to personal_info endpoint
- Do NOT take `data[0]` from ring_configuration (may be oldest, not most recent)
